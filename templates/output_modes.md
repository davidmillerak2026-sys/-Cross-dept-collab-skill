# 输出模式模板

根据用户请求选择一个主模式。未指定时，默认输出“统一事件包 + 跨部门行动项 + 系统动作卡 + 风险提示 + 待确认信息 + 同步消息”；只有维修、质量或系统落地需要时，再补充系统记录草案。

## 系统记录模式

用于维修、质量、EHS、换线、备件等现场事项。

```markdown
## 系统记录草案

- 系统记录标题：
- 场景分类：
- 优先级：
- 现场位置：
- 设备/产线/工位：
- 发生时间：
- 问题现象：
- 影响范围：
- 已采取措施：
- 初步判断：
- 建议下一步：
- 需要协同部门：
- 截止时间建议：

## 已确认事实

1.

## 合理推断

1.

## 风险与合规提示

- 质量风险：
- 生产影响：
- 数据/隐私注意：

```

```markdown

- 时间/地点：
- 参与角色：
- 背景摘要：
- 关键事实：

- 未解决争议：

| 行动项 | 负责人/责任方 | 协同方 | 截止时间 | 验收标准 | 风险 | 状态 |

```

## 审批说明模式

用于备件申请、停机申请、临时放行、整改延期、外协支持、资源协调。

```markdown
## 审批说明草稿

- 申请事项：
- 申请原因：
- 业务影响：
- 风险控制措施：
- 所需资源：
- 不批准的后果：
- 建议审批路径：
- 附件/证据清单：
- 待确认信息：
```

不要替代授权人给出最终批准、放行或安全许可结论。
