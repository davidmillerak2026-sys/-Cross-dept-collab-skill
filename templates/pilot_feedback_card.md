# 试用反馈卡模板

用途：审核通过后，收集制造业用户对 Skill 的真实试用反馈，用于证明“作品创新性和应用价值”。不得伪造反馈，不得诱导刷收藏、刷下载或夸大效果。

## 反馈收集原则

- 只收集角色、场景、使用前后差异和可公开评价，不收集身份证号、手机号、客户名称、内部订单号或系统账号。
- 如来自真实企业现场，必须先脱敏；截图中不得包含客户、员工、供应商、订单、价格、产线编号等敏感信息。
- 反馈只能写“试用者认为”“本次样例中节省”，不能写成全公司已实现收益。
- 未经允许不得公开试用者姓名、公司、头像、聊天记录或系统记录截图。

## 单次反馈卡

| 字段 | 填写说明 |
| --- | --- |
| feedback_id | 建议用日期 + 角色 + 场景，例如 `FB-20260616-quality-T06` |
| 试用角色 | 生产主管、设备维修、质量工程师、EHS、安全员、PMC、仓库/采购、数字化负责人 |
| 试用场景 | 对应 T01-T39 或用户自定义脱敏场景 |
| 输入类型 | 语音转写、巡检记录、会议纪要、已有系统记录、质量记录、计划变更 |
| 使用前流程 | 原本需要谁整理、用哪些系统、耗时多久、最容易漏什么 |
| Skill 输出 | 系统记录、跨部门待办、动作卡、复盘、审批说明、周报、同步消息 |
| 使用后价值 | 节省整理时间、减少漏项、提升责任清晰度、减少反复沟通、提高闭环质量 |
| 可量化指标 | 分钟/次、遗漏项减少、字段完整率、跨部门反馈时效、关闭前补字段数量 |
| 仍需人工确认 | 质量放行、安全许可、客户承诺、系统写入、根因验证、系统记录关闭 |
| 可公开引用短评 | 1-2 句话，必须脱敏，不写公司和个人敏感信息 |
| 证据材料 | 截图路径、试用记录、脱敏输出文件、反馈表路径 |
| 是否同意公开 | yes/no；默认 no |

## 快速访谈问题

1. 这类现场记录平时由谁整理？平均一次需要多久？
2. 哪些信息最容易漏：责任方、截止时间、验收标准、复检结论、系统回执还是审批人？
3. Skill 输出中哪些部分可以直接复制到企业 IM、OA、CMMS、QMS、EHS、SAP/ERP 或 PMC 排产表？
4. 哪些结论仍必须由人工或授权人确认？
5. 你愿意把它用于哪些真实场景：维修、质量、EHS、PMC、工程变更、日清会、周报？
6. 如果用于真实流程，最需要补哪一个系统对接或模板？
