# OCAS-skill 试用反馈收集表

用途：审核通过后，收集真实试用反馈，用于专家榜“创新性和应用价值”证据。所有反馈必须真实、脱敏、可追溯，不得购买、诱导或伪造。

## 推荐试用对象

| 角色 | 推荐用例 | 关注价值 |
| --- | --- | --- |
| 生产主管 | T01 / T36 / T38 / T39 | 是否能把现场异常转成跨部门待办和同步节奏 |
| 设备维修 | T01 / T25 / T34 / T35 | 是否能补齐检查、备件、试运行和关闭门禁 |
| 质量工程师 | T06 / T21 / T30 / T33 | 是否能守住复检、隔离、放行授权和冲突事实 |
| EHS/安全 | T11 / T12 / T38 | 是否能拒绝危险操作并保留整改证据 |
| PMC/计划 | T36 / T39 | 是否能看到产能、交期、物料和客户承诺边界 |
| 数字化负责人 | T21 / T28 / T39 | 是否能区分企业 IM 通知和正式系统回执 |

## 单次反馈记录

```markdown
### FB-YYYYMMDD-role-case

- 试用角色：
- 试用用例或场景：
- 使用前整理方式：
- 使用前估算耗时：
- Skill 输出是否可直接复制到企业 IM/OA/CMMS/QMS/EHS/SAP/PMC：
- 最有价值的输出：
- 仍需人工确认：
- 估算节省时间：
- 遗漏项是否减少：
- 1-5 分评分：
- 可公开短评：
- 是否同意公开脱敏短评：
- 证据截图或输出文件：
```

## 证据边界

- 没有用户授权时，只能汇总为匿名角色反馈。
- 没有实际计时数据时，只能写“估算节省”，不能写成已实现收益。
- 不能把收藏、下载或热度数据写成专家榜技术质量证明。
- 不能暗示 Skill 已自动放行质量、自动批准安全许可、自动关闭系统记录或自动写入外部系统。
