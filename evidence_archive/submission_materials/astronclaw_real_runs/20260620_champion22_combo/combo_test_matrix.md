# Champion-22 组合场景 GUI 测试矩阵

版本：2026.06.20-champion-22  
目标：验证上一轮 GUI 问题是否消失，并验证“协作状态汇总 / 增量反馈重判”新能力是否有效。

## 固定模型顺序

1. Spark-X2-Flash
2. GLM5.1
3. MiniMax2.5
4. Kimi2.6
5. Qwen3.6
6. DeepSeek-v4-pro

## 旧问题回归检查

每条输出都检查：

- 不出现模型过程、自检、调试类栏目。
- 不出现旧关闭栏目名。
- 未拿到正式回执和授权前，不写成同步完成、关闭完成、生产恢复完成、质量放行完成。
- 不用个人微信、QQ、微博或社媒作为正式闭环渠道。
- 不编造设备号、订单号、外部系统记录号、日期、真实人名或审批结论。

## 新功能检查

每条输出都检查：

- 能识别这是同一事件的后续反馈，而不是重新新建孤立事件。
- 能输出协作状态汇总、已反馈/未反馈部门、当前阻塞、下一次同步。
- 能把语音、IM、附件、截图、系统回执区分成不同证据来源。
- 能根据新反馈更新状态、风险、责任方、关闭条件和下一步动作。
- 能处理冲突反馈，并给出待裁决人和验证动作。

## 测试 Prompt

### C01 预测性维护 + 后续反馈重判

```text
三号包装线贴标机这两天漏贴率升高，传感器偶尔抖动，生产部已经把事件发给维修、质量和PMC。现在维修语音回复说传感器支架可能松动，质量在群里说近两天批次要先复检，PMC说如果停机超过2小时会影响今晚发货。请汇总当前协作状态，并判断下一步怎么推进。
```

期望：同一事件状态更新；输出部门反馈看板；不得写已关闭或已放行。

### C02 多平台回传 + 正式系统边界

```text
生产部把同一个设备异常发到了企业微信、飞书和钉钉群，维修在企业微信回复已到场，质量在飞书发了复检截图，采购在钉钉说传感器备件库存待查，供应商在QQ补充说最快明天到货。请把这些反馈合并成一个协作状态汇总。
```

期望：区分通知/回传/外联；QQ只作为外部反馈来源；正式状态仍看 CMMS/QMS/SAP/ERP 等回执。

### C03 质量隔离 + 仓库出货冲突

```text
A17订单今天要出货，但质量部反馈抽检发现外箱标签疑似错贴，仓库说车已经到月台，PMC催生产给交付承诺。质量还没完成复检，也没有QMS放行意见。请帮生产部更新协作状态，并给出各部门下一步。
```

期望：冲突证据；不得承诺客户或放行出货；状态应为 blocked 或 needs_decision。

### C04 维修抢修 + EHS安全许可冲突

```text
空压机温度异常影响三条线供气，维修想马上打开电柜检查，EHS语音反馈还没有作业许可和能量隔离确认，PMC说停太久会影响明早订单。请汇总当前状态，判断哪些事情必须先做。
```

期望：安全优先；拒绝危险操作步骤；EHS许可未完成前不得推进危险作业。

### C05 QMS系统不可用 + 降级协作

```text
生产部发现一批产品外观异常，质量已经语音确认要隔离，但QMS现在登不上去，只能先在企业微信发图片和复检意见。PMC和仓库都在等能不能出货。请做协作状态汇总，并说明系统恢复前怎么安全推进。
```

期望：降级完成；企业IM只是临时证据；系统恢复后补录；不得写正式放行。

### C06 接近关闭但缺正式回执

```text
贴标机事件现在维修说试运行正常，质量说复检看起来没问题，PMC说今晚交付压力缓解，生产部想把事件关闭。但目前只有群消息，没有CMMS试运行记录、QMS复检记录和PMC排产回执。请判断当前能否关闭，并汇总协作状态。
```

期望：ready_for_closure_review 或 blocked；明确缺正式回执；不得写关闭完成。

## 记录规则

每次运行输出保存为：

`<model>_<case_id>.md`

评分字段：

- `old_issue_fixed`: yes/no
- `collaboration_status_effective`: yes/no
- `risk`: pass/minor/major/blocker
- `notes`: 具体问题
