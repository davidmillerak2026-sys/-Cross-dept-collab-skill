# OCAS-skill 参赛策略与包装

## 推荐参赛作品

作品名称：工业现场跨部门协同工作 Skill

SkillHub 名称：`industrial-cross-department-collaboration`

赛题链接：https://challenge.xfyun.cn/topic/info?type=OCAS-skill

截止时间：2026 年 8 月 27 日 17:00

## 为什么匹配

OCAS-skill 的核心是“办公协同助理 Skill”，重点关注会议管理、文档处理、日程任务、跨部门协同和流程审批。我们的项目如果直接讲工业 AR/边缘模型，会显得离“办公协同”偏远；但把一线语音、巡检和故障记录转成生产部牵头的统一事件包、跨部门行动项、系统动作卡、审批/复盘/周报和必要的系统记录草案，就正好落在办公协同链路。

## 包装口径

一句话：

> 面向制造企业现场协同的 AI Skill，将维修、质量、安全、换线和交接记录自动整理为统一事件包、跨部门行动项、系统动作卡、复盘纪要和审批说明；必要时再生成系统记录草案。

不要主打：

- 硬件平台
- AR 眼镜参数
- 某个模型名称
- “替代工程师判断”

要主打：

- 信息不丢失
- 行动项可追踪
- 跨部门责任清晰
- 安全和质量风险显式提示
- 可对接 CMMS / MES / QMS / EHS / OA

## 评审卖点

1. 真实痛点：工业现场信息常停留在语音、微信群、纸质点检表和个人经验中，难以进入办公协同流程。
2. 可落地：输出就是事件包、行动项、系统动作卡、审批、纪要和周报，必要时能落到系统记录系统，不需要评委理解复杂硬件系统。
3. 鲁棒性：Skill 明确区分“已确认事实、合理推断、待确认信息”，减少误报和编造。
4. 安全合规：对人身安全、质量放行、客户信息和真实数据脱敏有明确约束。
5. 可扩展：后续可以接入 AR 端、边缘 Agent、CMMS、MES、QMS、EHS、企业微信/钉钉/飞书。

## 建议演示脚本

演示 1：现场语音转维修系统记录

- 输入一段设备异常语音转写。
- 输出系统记录草案、待办表、风险提示、待确认信息。

演示 2：质量异常转跨部门协同

- 输入质量、生产、仓库、销售的碎片化描述。
- 输出隔离、复检、客户同步、放行约束等待办。

演示 3：一周记录转管理周报

- 输入 5 条现场事件。
- 输出管理层关注项和下周重点。

## 需要补充的真实材料

- 产品/系统截图：AR 端或系统记录端界面均可。
- Demo 视频：建议 60-120 秒，展示语音输入到结构化系统记录。
- 脱敏客户案例：不要出现真实客户名、真实设备编号或个人信息。
- 技术说明：边缘采集、语音识别、多模态记录、Agent 编排、审计卡片。

## 不建议写法

- “我们的系统可以自动判断根因并直接下发维修指令。”
- “可以绕过现有审批流程，提高效率。”
- “已在某客户现场大规模使用”但没有证据。
- 放入真实 API Key、客户名称、设备私密数据。
