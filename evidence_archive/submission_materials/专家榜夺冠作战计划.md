## 夺冠判断

不能把目标理解为“提交一个 Skill”。专家榜按 100 分评审，冠军需要证明三件事：

1. 作品在 SkillHub 审核通过，并在 AstronClaw 中能稳定部署调用。
2. 作品强贴合“办公协同助理 Skill”赛题，不跑偏成工业硬件或平台广告。
3. 作品具备明确商业价值、工程规范、评审证据和可传播性。

## 作品定位

作品名：工业现场跨部门协同工作 Skill  
Skill 名称：`industrial-cross-department-collaboration`

一句话：

> 面向制造企业工业现场跨部门协同断层的 AI Skill，将设备故障、巡检记录、质量异常、EHS 隐患、换线确认和班组交接整理成统一事件包、跨部门行动项、系统动作卡、复盘纪要、审批说明和管理周报；必要时再生成系统记录草案。

核心差异化：

- 不是通用会议纪要，也不是工业硬件展示。
- 主打“现场事实进入办公协同流程”。
- 把已确认事实、合理推断、待确认信息分离，降低误判。
- 每个行动项带责任方、依据、截止时间、验收标准。
- 安全和质量场景保守处理，不替代授权人判断，不绕过 SOP。

## 冠军版交付物

### 1. Skill 包

必须包含：

- `SKILL.md`
- README
- 3-5 个示例
- 2-3 个 JSON Schema / 模板
- 版本记录
- 测试用例说明

升级目标：

- 加入“场景路由表”：maintenance / quality / safety / changeover / meeting / handover
- 加入“输出质量检查清单”：责任方、依据、验收标准、待确认信息
- 加入“异常输入处理”：输入过短、信息冲突、缺少时间/设备/责任方
- 加入“评审友好说明”：为什么贴合办公协同赛题

### 2. 测试证据包

至少准备 39 个测试用例：

- 5 个设备维修
- 4 个质量异常
- 3 个 EHS 安全
- 3 个换线/SKU 切换
- 3 个会议复盘
- 2 个交接班/周报
- 2 个现场执行与系统记录数据质量
- 4 个生产部牵头的 PMC、工程、跨部门日清和部门系统流转协同

每个用例记录：

- 输入文本
- 期望输出类型
- 实际输出截图
- 是否含责任方
- 是否含依据
- 是否含待确认信息
- 是否有不当编造
- 是否触发安全/质量提示

### 3. Demo 视频

必须做 60-90 秒版本。

结构：

1. 痛点：现场信息散落，办公协同成本高
2. 输入：一段设备/质量异常现场记录
3. 输出：系统记录草案
4. 输出：跨部门待办
5. 输出：风险和待确认信息
6. 行动：SkillHub 收藏、下载、试用

### 4. 评审材料

准备一份 PDF/PPT：

- 赛题匹配说明
- 痛点和应用场景
- Skill 结构
- 输入输出案例
- 稳定性测试结果
- 安全合规说明
- 商业应用价值
- SkillHub 链接和热度数据

### 5. 热度榜运营

目标不是刷量，而是真实传播。

必做：

- 审核通过当天首发朋友圈/微信群
- 发布技术文章
- 发布 Demo 视频
- 私聊制造业/AI/数字化相关朋友真实试用
- 每日记录收藏量、下载量、排名、反馈

禁止：

- 脚本刷量
- 购买收藏下载
- 让无关账号机械点击
- 用红包/返利诱导点击

## 决策原则

- 所有新增内容必须服务专家榜评分。
- 宁可减少炫技，也要让办公协同价值更清楚。
- 证据优先于口号。
- 真实试用优先于泛流量。
- 安全合规不冒险。
