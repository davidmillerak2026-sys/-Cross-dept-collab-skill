# 办公协同消息模板

## 系统记录同步

```text
【系统记录同步】{产线/设备} 于 {时间} 发生 {问题现象}，当前影响 {影响范围}。已采取措施：{已采取措施}。请 {责任方} 在 {截止时间} 前完成 {待办事项}，验收标准：{验收标准}。待确认：{待确认信息}。
```

## 质量异常同步

```text
【质量异常】{批次/SKU} 发现 {异常现象}，影响范围 {数量/区域}。建议先隔离并完成复检，未经质量授权不建议放行。请质量确认复检标准，生产/仓库配合追溯，销售暂缓对外承诺。
```

## EHS 整改同步

```text
【EHS整改】现场发现 {隐患描述}，风险等级 {优先级}。请先隔离风险并按 SOP 处理，由有资质人员确认整改结果。待办：{整改动作}。验收标准：{验收标准}。
```

## 会议复盘同步

```text
【复盘结论】本次复盘围绕 {主题}。已确认事实：{事实摘要}。决策点：{决策点}。行动项：{责任方+动作+截止时间}。待确认：{待确认信息}。
```

## 跨平台协同通知

适用于 AstronClaw 已配置的企业微信/企微、飞书、钉钉、微信和邮件等协作入口；也可在企业授权、脱敏和审计留痕满足条件时，转成 QQ、微博、客服系统或公众平台的外联草稿。正式闭环仍以 MES/CMMS/QMS/EHS/SAP/ERP/OA/PMC 回执为准。未提供渠道开通证据时，连接器状态写 `needs_configuration`，不得声称推送成功。

```text
【跨部门协同】事件：{事件摘要}。生产部 Owner：{Owner角色}。当前影响：{生产/质量/安全/交付/物料影响}。请 {部门/角色} 在 {截止时间} 前反馈：{反馈字段}。系统回填：{目标业务系统}。待确认：{缺失字段}。下一次同步：{时间/触发条件}。
```

## 多端协作动作卡

```text
platform：{企业微信/企微/飞书/钉钉/微信/邮件/授权外联平台}
connector_status：{ready / needs_configuration / optional_authorized_connector}
recipient_scope：{角色/群组/授权外部联系人}
message_template：{可复制短消息}
feedback_contract：{必须回传字段}
business_record_backlink：{MES/CMMS/QMS/EHS/SAP/ERP/OA/PMC 回填字段}
fallback_channel：{邮件/OA/电话/客服系统}
boundary：该渠道只负责通知、催办、轻量确认或反馈收集，正式状态以业务系统回执和授权人为准。
```

## 外联反馈收集

```text
【反馈收集】我们正在核实 {事件/批次/订单影响}，请在 {截止时间} 前反馈 {需要反馈的字段}。本消息仅用于信息收集，最终处理状态以企业正式系统和授权通知为准。请勿在回复中发送敏感个人信息、账号凭证或未脱敏附件。
```
