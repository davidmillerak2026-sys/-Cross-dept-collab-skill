# Industrial Cross Department Collaboration

面向工业现场的跨部门协同工作 Skill。它帮助生产部把现场异常、部门反馈、系统回执和附件证据整理成统一事件包、协作状态汇总、部门反馈合同、正式系统动作卡、升级决策包和关闭条件。

## 适用场景

- 设备故障、非计划停机、重复异常、备件等待。
- 质量异常、批次隔离、复检、返工、放行、客户投诉。
- EHS 隐患、作业许可、能量隔离、安全整改。
- PMC 排产、交期冲突、加班调产、发货风险。
- QMS/MES/ERP 等系统不可用时的应急留痕和补录。
- 企业微信、飞书、钉钉、微信、邮件等渠道中的持续反馈汇总。

## 核心能力

- 统一事件包：把现场描述转成可追踪的事件信息。
- 证据分层：区分已确认事实、合理推断、待确认信息和冲突反馈。
- 部门反馈合同：明确每个部门需要回传的字段、系统和时限。
- 系统动作卡：区分通知渠道和 MES/CMMS/QMS/EHS/SAP/OA/PMC 等正式业务系统。
- 协作状态汇总：对后续语音、截图、附件、系统回执持续更新事件状态。
- 风险与升级：在安全、质量、生产、交付、物料、成本受影响时生成影响评分板和管理升级决策包。
- 关闭条件检查：没有正式回执和授权证据时，不把事件写成已关闭、已放行或已恢复。

## 使用方式

直接用自然语言描述工业现场问题即可，例如：

```text
三号包装线贴标机这两天漏贴率明显升高，传感器偶尔抖动。
维修只说再观察，生产担心突然停线。请帮我组织跨部门协同。
```

如果是事件过程中各部门继续反馈，可以这样输入：

```text
维修在企业微信说已经到场，质量在飞书发了复检截图，
采购在钉钉说传感器备件库存待查，供应商 QQ 回复最快明天到货。
请汇总当前协作状态，并告诉我下一步怎么推进。
```

## 业务边界

- 企业微信、飞书、钉钉、微信和邮件用于通知、催办、轻量确认和附件回传。
- MES、CMMS、QMS、EHS、SAP/ERP、OA、PMC/APS 承担正式业务记录和授权闭环。
- QQ、微博、供应商聊天或个人渠道只能作为外部信息来源，不能替代正式系统回执。
- 未提供系统回执、授权确认或附件证据时，只能输出待确认、处理中、需授权或待回执。

## 包体边界
