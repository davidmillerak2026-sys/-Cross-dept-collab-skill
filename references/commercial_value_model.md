# 商业价值量化模型

用途：把“工业现场到办公协同”的价值表达成可审计、可试用验证的指标，支撑专家榜“作品创新性和应用价值”。该模型只用于估算和试用记录，不得在没有证据时宣称已实现收益。

## 价值假设

1. 减少整理时间：一线记录、会议纪要、工单草稿和跨部门待办由人工整理改为 Skill 生成草稿。
2. 减少漏项：责任方、依据、截止时间、验收标准、待确认信息和系统回执被显式列出。
3. 减少反复沟通：生产部、维修、质量、工程、EHS、PMC、仓库/采购围绕同一事件包反馈。
4. 提高闭环质量：关闭工单前检查故障码、原因码、备件、工时、试运行、复检和附件。
5. 降低合规风险：质量放行、安全许可、客户承诺、系统写入和知识发布进入授权门禁。

## 试用量化指标

| 指标 | 定义 | 记录方式 |
| --- | --- | --- |
| time_saved_minutes | 使用前人工整理耗时 - 使用后整理/校对耗时 | 试用者估算或实际计时 |
| missing_field_reduction | 使用前遗漏字段数 - 使用后遗漏字段数 | 对照工单/待办字段检查 |
| handoff_clarity_score | 责任方、截止时间、验收标准是否清晰，1-5 分 | 试用反馈卡 |
| system_traceability_score | 是否标出 MES/CMMS/QMS/EHS/SAP/OA/PMC 回执边界，1-5 分 | 输出人工复核 |
| unsafe_claim_count | 输出中出现越权放行、危险操作、伪造系统状态的次数 | 应为 0 |
| close_ready_rate | 工单关闭前数据质量门禁通过比例 | AstronClaw 或真实试用记录 |
| production_loss_units | 是否把停线分钟、少产数量或待复检数量讲清楚 | MES/试用反馈 |
| delivery_risk_level | 是否清楚标出交付风险等级和决策截止时间 | PMC/APS/试用反馈 |
| blocked_close_reason_count | 当前阻止闭环的关键缺口数量 | 输出人工复核 |

## 估算公式

```text
单次协同节省时间 = 使用前整理分钟 - 使用后校对分钟
月度节省时间 = 单次协同节省时间 * 月度类似事件数
漏项改善 = 使用前平均遗漏字段数 - 使用后平均遗漏字段数
闭环质量得分 = 责任清晰度 + 系统可追踪性 + 合规边界清晰度
管理决策提速 = 管理层反复追问轮次减少 + 决策截止时间被提前显式化
```

## 示例记录

```markdown
试用场景：T39 部门沟通与系统流转
使用前：生产主管需要在群消息、MES、QMS、维修反馈之间手工整理，约 20 分钟。
使用后：Skill 生成统一事件包、部门反馈合同、影响评分板和系统回执清单，人工校对约 7 分钟。
估算节省：13 分钟/次。
仍需人工确认：质量复检、维修试运行、EHS 许可、PMC 交期承诺。
结论边界：这是单次试用估算，不代表全公司已实现收益。
```

## 专家评审表达

可以说：

> 本 Skill 的商业价值不是替代业务系统，而是降低现场信息进入办公协同流程的整理成本和漏项风险。我们用试用反馈卡记录节省时间、字段完整度、系统回执可追踪性和人工授权边界。

> 当事件升级到管理层时，本 Skill 还能用影响评分板和管理升级决策包减少反复问答，让生产、质量、安全和 PMC 围绕同一组事实做决策。

不能说：

- 已经为企业节省多少成本，除非有真实授权数据。
- 已经自动关闭工单、自动质量放行、自动调整排产。
- 已经接入某企业生产系统，除非有真实回执和授权证明。
