# AstronClaw 企业部门协同 10 场景专项实测包

用途：审核通过后，把以下 S01-S10 提示词逐条复制到 AstronClaw，用于证明本 Skill 能处理真实企业生产部牵头的跨部门协同闭环。

专项场景数量：10

通过条件：输出必须能区分通知层、正式系统留痕、部门反馈、授权边界、关闭门禁和关闭后知识沉淀；不得声称外部系统已自动完成。
对高影响场景，还必须体现影响评分板、管理升级决策包、决策截止时间和若不决策的后果。

## S01 贴标机漏贴导致非计划停机

证据焦点：生产部发现、通知层、正式系统链路、部门反馈、闭环门禁、关闭后知识沉淀  
场景：`enterprise_flow`

复制到 AstronClaw 的提示词：

```text
使用 industrial-workorder-collaboration 处理下面的企业现场协同场景。请按 Skill 规则输出可执行结果，必须覆盖：

1. 生产部发现什么，以及如何建立统一事件包。
2. 企业微信/飞书/钉钉/邮件如何通知、催办和轻量确认。
3. MES、CMMS、QMS、EHS、SAP/ERP、OA、PMC/APS/排产表分别如何留痕和回执。
4. 各部门必须反馈什么字段，哪些缺口会阻塞闭环。
5. 生产部如何汇总、升级和判断能否关闭。
6. 知识库/SOP 只能在事件关闭后生成候选沉淀，不能参与最初紧急流转。
7. 因为这里同时涉及停线、质量隔离和交付风险，额外输出影响评分板和管理升级决策包，写清停线/少产、交付风险、阻塞关闭原因数、决策截止时间和若不决策的后果。

专项场景：

贴标机漏贴导致非计划停机

### 生产部发现

三号包装线贴标机在 14:20 后连续漏贴，换标签卷后无改善，现场少产风险扩大。

### 生产部发起

- 在 MES 记录停线开始时间、产线、SKU、订单、少产估算和目标恢复窗口。
- 生成统一事件包：`20260616-L3-labeler-unplanned-stop`。
- 在企业微信/飞书/钉钉/邮件同步维修、质量、EHS、PMC、仓库/采购，要求 15 分钟内首轮反馈。

### 正式系统链路

| 系统 | 动作 | 必须回执 |
| --- | --- | --- |
| MES | 记录停线、少产、恢复窗口 | 停线时长、影响订单、恢复目标 |
| CMMS | 建立维修工单并派工 | 到场时间、候选原因、备件、工时、试运行结果 |
| QMS | 建立隔离和复检记录 | 隔离箱数、复检结论、放行或返工授权 |
| EHS | 判断是否需要能量隔离 | 风险等级、许可状态、隔离确认 |
| SAP/ERP | 查询标签卷、传感器备件和订单影响 | 库存、到料时间、交付影响 |
| PMC/APS | 评估调产、加班或插单 | 新排产方案、客户承诺边界 |
| OA | 若停线超阈值，生成异常报告 | 审批状态、升级意见 |

### 部门反馈

| 部门 | 反馈内容 | 生产部如何处理 |
| --- | --- | --- |
| 维修 | 传感器偏位为候选原因，需调整后试运行 20 分钟 | 更新事件包，未试运行前不写已修复 |
| 质量 | 32 箱隔离，抽检未完成 | 维持待复检，不允许恢复全量入库 |
| EHS | 需要断电挂牌后开机罩检查 | 将作业许可作为恢复前门禁 |
| PMC | 若 15:30 前未恢复，需切换二号线并安排加班 | 在下一次同步前准备调产方案 |
| 仓库/采购 | 备件库存 1 个，采购周期 2 天 | 若备件消耗，触发 SAP/ERP 补货 |

### 管理升级触发

- 当停线、质量隔离和交付风险同时存在时，补输出影响评分板，量化停线分钟、少产数量、交付风险等级、阻塞关闭原因数。
- 同步生成管理升级决策包，写明决策截止时间、若不决策的后果，以及继续抢修/切换产线/加班补产等候选方案。

### 闭环门禁

- CMMS 有维修措施、工时、备件和试运行结果。
- QMS 有复检结论和放行/返工授权。
- EHS 有能量隔离解除记录。
- MES 更新实际恢复时间和少产数量。
- PMC 确认交期或调产方案。
- 事件关闭后再生成知识库/SOP 候选条目。
```

### 部门反馈

| 部门 | 反馈内容 | 生产部如何处理 |
| --- | --- | --- |

### 管理升级触发

- 当设备、质量、待料、EHS 和交付多事项并存时，补输出影响评分板，量化每个事项的影响等级、阻塞关闭原因数和决策截止时间。
- 生成管理升级决策包，写明若不决策的后果，以及先处理顺序、资源倾斜和升级对象。

### 闭环门禁

- OA 行动项有责任人、截止时间和状态。
- 各业务系统分别有回执，不能只看会议纪要。
- 质量、安全、维修和交付风险全部确认后，生产部才可关闭日清会事项。
- 关闭后沉淀“日清会跨部门行动项模板”候选知识。
