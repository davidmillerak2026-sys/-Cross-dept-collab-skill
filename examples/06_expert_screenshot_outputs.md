#### 端到端流转

1. 生产部先在 MES 留下停线事实和产量影响，再用企业 IM 发出同一份事件包。
2. 维修在 CMMS 接收或更新工单，回传候选原因、维修措施、备件需求和试运行结果。
3. 质量在 QMS 建立隔离/复检记录，复检未完成前不能放行 32 箱，也不能作为恢复生产依据。

5. PMC 结合 MES 停线、QMS 放行、SAP/ERP 备件或包材状态，判断是否调产、加班或升级交期风险。

- 不写“系统已同步成功”，除非有真实系统回执。

## T39 部门沟通与系统流转专项问答

### 场景分类

- 主场景：`department_flow`
- 次场景：`maintenance / quality / safety / pmc`
- 优先级：`P1`

### 总体回答

这类事件不能靠 Facebook、个人社媒或零散聊天记录完成正式闭环。生产部应作为事件 Owner，用统一事件包把现场事实、生产影响、质量风险、待确认信息、下一次同步时间和升级路径发给各部门。企业微信、飞书、钉钉和邮件用于通知、催办、会议邀约；MES、CMMS、QMS、EHS、SAP/ERP、OA、PMC/APS 用于业务留痕、审批和系统回执。

### 统一事件包

| 字段 | 当前建议内容 |
| --- | --- |
| event_id | 20260616-L3-labeler-leak-stop |
| 生产部 Owner | 三号包装线当班生产负责人 |
| 已确认事实 | 贴标机频繁漏贴，存在非计划停机风险 |
| 合理推断 | 设备或标签触发链路异常，仅作为候选方向 |
| 待确认 | 设备号、停线开始时间、受影响批次、维修到场时间、质量复检结论、PMC 交期影响 |
| 下一次同步 | 维修初检后 15 分钟内，或生产恢复窗口前 |
| 升级路径 | 超时未反馈或风险扩大时升级给生产经理、维修主管、质量负责人、PMC 负责人 |

### 渠道与系统分层

| 层级 | 工具/系统 | 负责什么 | 不负责什么 |
| --- | --- | --- | --- |
| 通知层 | 企业微信/飞书/钉钉/邮件 | 通知、催办、抄送、会议邀约 | 不作为最终维修/质量/安全闭环证据 |
| 非正式渠道 | Facebook/个人社媒 | 不建议用于企业正式协同 | 不做业务留痕、不承载质量/安全/工单闭环 |
| 生产层 | MES | 停线、产量、节拍、恢复窗口 | 不做维修关闭或质量放行 |
| 维修层 | CMMS | 维修工单、派工、备件、工时、试运行 | 不做质量判定 |
| 质量层 | QMS | 隔离、复检、放行申请、CAPA | 不由生产部直接批准放行 |
| 安全层 | EHS/OA | 作业许可、能量隔离、挂牌、整改闭环 | 不绕过 SOP 或资质要求 |
| 物料/订单层 | SAP/ERP | 备件、包材、库存、采购、订单影响 | 不替代现场执行确认 |
| 计划层 | PMC/APS | 排产、产能、交期、加班、插单 | 不替代质量/安全授权 |

### 部门反馈合同

| 部门 | 必须反馈 | 正式系统回执 | 缺失时状态 |
| --- | --- | --- | --- |
| 生产部 | 停线影响、少产数量、恢复窗口、下一次同步时间 | MES 记录号或生产日报/OA 纪要 | `needs_confirmation` |
| 维修 | 到场时间、候选原因、维修措施、备件、工时、试运行结果 | CMMS 工单号和试运行记录 | `not_close_ready` |
| 质量 | 隔离范围、复检结论、放行/返工授权人 | QMS 复检记录和授权意见 | 不得写质量已放行 |
| 工程/工艺 | 参数版本、工艺变更依据、试产验证边界 | OA/工程变更记录或 MES 版本备注 | 不得写变更已生效 |
| EHS/安全 | 许可状态、隔离措施、宣导/整改验证 | EHS/OA 许可或整改记录 | 不得开始高风险作业 |
| PMC | 交期风险、调产/加班方案、客户承诺边界 | PMC/APS/SAP PP 回执 | 不得承诺客户交期 |

### 端到端流程

1. 生产部先在 MES 或生产记录中登记停线/异常事实，形成统一事件包。
2. 生产部把事件包摘要发到企业微信/飞书/钉钉或邮件，要求各部门按截止时间回传。
3. 维修在 CMMS 接工单并回传检查、措施和试运行；质量在 QMS 做隔离和复检；安全在 EHS/OA 判断许可；PMC 在排产系统评估交期；仓库/采购在 SAP/ERP 查询备件和包材。
4. 各部门反馈回同一事件包，生产部汇总已完成、待确认、需授权、阻塞和升级事项。
5. 关闭门禁：缺维修试运行、质量复检、EHS 许可、PMC 交期确认、SAP/ERP 物料回执或系统记录号时，只能保持处理中/待确认，不能写已同步、已恢复生产、质量已放行或工单已关闭。

### 可复制同步消息

生产部已发起三号包装线贴标机漏贴事件协同。当前只确认存在漏贴和非计划停机风险，根因、受影响批次、维修措施和复检结果仍待确认。请维修通过 CMMS 回传到场和试运行，质量通过 QMS 回传隔离/复检，安全确认是否需要 EHS/OA 许可，PMC 结合 MES 与 SAP/ERP 信息评估交期影响，仓库/采购回传备件和包材状态。未取得系统回执和授权前，不写已恢复生产、质量已放行或工单已关闭。

### 禁止输出

- 不把 Facebook 或个人社媒写成正式闭环渠道。
- 不用聊天确认替代 CMMS/QMS/EHS/SAP/OA 等业务系统回执。
- 不写“已同步系统”“已关闭”“已恢复生产”“质量已放行”，除非用户提供真实系统回执。
